## ⛔ P0-BLOCKER (zuerst lösen) — CI ist für ALLE PRs rot

Die Pflicht-Checks **`build`** und **`Migration validation`** scheitern auf **jedem** PR
bereits beim Schritt *„Prepare all required actions"* — vor jeder Datei-Verarbeitung:

```
The action actions/checkout@v4 is not allowed ... because all actions
must be pinned to a full-length commit SHA.
```

**Ursache:** Repo-/Org-Policy verlangt SHA-gepinnte Actions; die Workflows
(`.github/workflows/*.yml`) referenzieren noch Tags (`actions/checkout@v4`,
`actions/setup-node@v4`, …).

**Konsequenz:** „Mergen nach CI-grün" ist **physisch unmöglich**, bis die Workflows
auf Commit-SHAs gepinnt sind. **Das ist der eigentliche Engpass des gesamten PR-Staus** —
nicht die einzelnen Feature-PRs. Empfehlung: einen kleinen, isolierten Workflow-PR
(alle `uses:`-Refs → SHA) **vor** allem anderen mergen, dann die Feature-PRs rebasen.
