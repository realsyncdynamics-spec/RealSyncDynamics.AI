# 40_security

Security process documentation. Each document describes a
process — detection, response, hardening — in operational
terms. The folder does not contain:

- Live threat intelligence (separate, restricted system)
- Credentials, keys, or secrets of any kind
- Customer-specific security data

| Document             | Purpose                                          |
|----------------------|--------------------------------------------------|
| `incident-response.md` | Detection, classification, containment, GDPR breach handling |

New security documents start from a runtime-spec or compliance
template adapted to security context. There is no dedicated
security template yet.
