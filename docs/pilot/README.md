Operative Marktvalidierung statt weiterer Positionierung.
Ein Use Case, ein Workflow, ein bezahlter Pilot.

> **Status:** Planungsdokument. Bindet Scope für die nächsten 90 Tage.
> Änderungen an Scope hier → bevor neuer Code dazukommt.

## Use-Case-Entscheidung

**Option A — Multi-Website Governance für Datenschutz-Agenturen.**

Begründung (kurz): kürzester Sales-Zyklus, Mandanten-Multiplier (1 Agentur = 20–50 Sites),
und der bestehende Stack (Cookie-Scanner, Audit-Result-Pipeline, `realsync-cli`,
`runtime_events`) deckt den MVP-Workflow zu ~70 % ab. B (AI-Chatbot-Governance) und
C (reines Vendor/Tracker-Monitoring) sind Folge-Use-Cases, nicht parallel.

## Verkaufswinkel — ein Satz

> Wir helfen Datenschutz-Agenturen, 20–50 Mandanten-Websites regelmäßig auf
> Tracker-, Vendor- und AI-Risiken zu prüfen — mit priorisierten Findings und
> exportierbarer, kryptographisch verifizierbarer Evidence.

Nicht: „Wir machen AI-Governance."
Nicht: „Operational Governance Runtime Infrastructure" (Kategorie-Sprache nur
auf Plattform-Seiten, nicht in Pilot-Outreach).

Infrastrukturell, technisch glaubwürdig, nüchtern. Erlaubt: *runtime-observable,
evidence-backed, replayable, tenant-isolated, deterministic verification*.
