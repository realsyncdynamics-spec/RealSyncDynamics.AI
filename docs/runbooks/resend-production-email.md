# Resend Production Email Runbook

> Pairs with `docs/runbooks/p0-pilot-blockers.md` (P0-1).

## Goal

Transactional mail delivery for:

- Welcome (`welcome-email`)
- Audit-report-ready (`audit-report-email`)
- Newsletter DOI (`newsletter-confirm`, `newsletter-subscribe`)
- Weekly audit re-check digest (`audit-recheck-weekly`)
- Daily digest (`daily-digest`)

## Required

- Sending domain verified in Resend dashboard: `realsyncdynamicsai.de`
- SPF record on the domain (TXT, `include:_spf.resend.com`)
- DKIM record (CNAME / TXT generated by Resend during domain verification)
- DMARC record (TXT, `v=DMARC1; p=quarantine; rua=mailto:dmarc@realsyncdynamicsai.de`)
- `resend_api_key` in `vault.secrets` (provisioned via `vault-set-secret` Edge Function)
- Sending key permission: **Sending access only** — never full-access in production

## Recommended sender

```
RealSyncDynamics.AI <noreply@realsyncdynamicsai.de>
```

For receipt-required mail (e.g. DPO contact), use `dpo@realsyncdynamicsai.de` with a real inbox.

## Failure modes

### Domain not verified
Resend rejects with `domain_not_found`. The welcome flow surfaces no error to the user (mail send is fire-and-forget) but logs `[welcome-email] resend 400`.

**Fix**: complete domain verification in Resend dashboard → Domains → Add → follow DNS instructions.

### API key missing
Edge Function logs `RESEND_API_KEY missing from env and vault`.

**Fix**: provision `resend_api_key` via `vault-set-secret`.

### SPF / DKIM missing
Mail delivers but goes to spam. Recipient mail server uses sender authentication for trust.

**Fix**: add SPF + DKIM records as instructed by Resend's domain wizard. DNS propagation 5–60 min.

### From-address not on verified domain
Resend rejects with `address_not_authorized`.

**Fix**: change the function's `from` to the verified domain. Currently hardcoded in each Edge Function — `_shared/mailer.ts` for the canonical value.

## Rotation

Rotate every 90 days. Calendar reminder mandatory. Procedure:

1. Generate new key in Resend dashboard with **Sending access** + Domain restriction `realsyncdynamicsai.de`
2. Provision new key via `vault-set-secret` (overwrites the existing vault entry)
3. Smoke-test welcome flow
4. Revoke old key in Resend dashboard
5. Update key-rotation log (`docs/security/key-rotation-log.md` — create if missing)

## Verification

```sql
-- Confirm resend key exists in vault (without exposing value)
SELECT name, length(secret) AS len, updated_at
FROM vault.secrets
WHERE name = 'resend_api_key';
```

Expect `len ≥ 30` and `updated_at` within last 90 days.
