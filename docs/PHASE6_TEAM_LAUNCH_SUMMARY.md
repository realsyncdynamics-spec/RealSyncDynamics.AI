### Stream 3️⃣: Engineer Onboarding Program (12-week certification track)

**Week 1-2:** Foundation & Setup

**Week 3-4:** Supervised Feature Implementation

- 2-3 moderate-complexity features shipped

**Week 9-10:** Capstone Ownership

- Code quality assessment

- 360-degree feedback collection
- Practitioner level certification awarded

- Ensures consistent quality and pattern adoption

## FAQ

**Q: What if I can't make the retrospective?**  
A: Attendance is expected and important for team cohesion. If impossible, notify the Engineering Lead immediately and catch up on notes.

**Q: What if the dashboard isn't ready by Friday?**  
A: Deploy what's working (even partial). Real-time health is highest priority. Weekly performance can follow Week 2-3.

**Q: What if the engineer isn't ready by Friday?**  
A: That's OK. Week 1 checklist is ambitious but flexible. The goal is foundation-building, not speed. Adjust pace as needed.

**Q: How do I get the dashboard link?**  
A: Posted in #engineering Friday morning before 10 AM UTC. Bookmark it for daily standup use Monday.

**Q: What happens with improvements we identify?**  
A: Each improvement gets a GitHub issue with owner and due date. Progress tracked weekly in retrospectives. Completed improvements celebrated.

**Q: Will there be training on how to use the dashboard?**  
A: Yes. DevOps lead conducts 15-min interpretation guide Friday afternoon. Recording available for those who miss it.

**Q: Is this permanent? What if we decide to change approach?**  
A: Phase 6 is the operational framework for sustainable scaling. Retrospectives are the mechanism for continuous refinement. We'll adjust based on what we learn.
